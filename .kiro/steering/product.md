# プロダクト Steering

## 目的

NTF YAML Editor は、Nablarch Testing Framework のテストデータを YAML として扱うための VS Code 拡張および CLI ツールチェインである。

このプロダクトの目的は、
- ExcelベースのテストデータのYAML移行
- 移行後の、YAMLテストデータのローカル編集やレビューの支援
- NTF テストデータを汎用 YAML テキストとしてではなく、表としてレビュー・編集できるようにする
- テストデータ破壊のガードレール

## 利用者

- Excel から移行した NTF YAML を確認・編集したいテストデータ編集者。
- YAML 移行後も、元の表の概念を失わずにテストデータを扱いたい開発者。
- AI が編集したテストデータを共有・コミット前に確認したいレビューイ。
- テストデータ変更をセル、行、ブロック、シート、ファイル単位で理解したいレビュワー。

## プロダクト原則

- NTF YAML を汎用 YAML 文書ではなく、構造化されたテストデータとして扱う。
- Excel 由来の表の概念を、YAML 移行後の編集・レビューでも保つ。
- 賢い推測よりも、レビュー可能で決定的な変換を優先する。
- テストデータの破壊や意図しない広範囲変更は、レビューしづらくなる前に見えるようにする。
- ローカル開発ワークフローを第一にする。CI 連携や PR 自動化は基本のプロダクト責務にしない。

## 現在の機能

- YAML ファイルを VS Code カスタムエディタの表エディタとして開く。
- トップレベル YAML キーをシートとして表現する。
- ファイル指定を持つ `VARIABLE` / `FIXED` 系ブロックをファイル系表として編集・レビューする。
- `VARIABLE` / `FIXED` 系以外の NTF block を通常 table として編集・レビューする。
- NTF YAML の構造解釈に基づいて表示・編集・diff を扱う。
- CLI で convert、lint、format を提供する。

## プロダクトの方向性

次の主要な方向性は、YAML 移行後のローカル編集とレビューを支える構造化されたセル差分である。

セル差分では、利用者が NTF YAML の変更を汎用テキスト差分ではなく、表のセル・行・ブロック・シート・ファイルの変更としてレビューできるようにする。これは、AI や手作業によるテストデータ編集が、意図した範囲に収まっているかを確認するためのガードレールでもある。

SCM diff は VS Code の diff editor が左右ペインを管理する。Cell Diff と HTML レポートは split / unified 表示を持つ。
ビュー構成は異なるが、表示意味、diff 意味、表コンポーネントの基本仕様は共有する。
HTML レポート生成はローカルで行い、レビュー直前の比較対象を明示して生成する。

セル差分の具体的な決定事項は `docs/cell-diff-design.md` を正とする。
