# ファイル系 table 表示・差分仕様

## 目的

NTF のファイル系ブロックを、Excel 由来の表構造として読みやすく編集・レビューできるようにする。
通常エディタ、SCM diff、cell diff、HTML レポートで、ファイル系 table の見え方と差分の意味を揃える。

## 対象

対象は次のファイル系ブロックとする。

- `SETUP_VARIABLE`
- `EXPECTED_VARIABLE`
- `SETUP_FIXED`
- `EXPECTED_FIXED`

`SETUP_FIXED` / `EXPECTED_FIXED` は、`VARIABLE` と同じ row 配列形式のファイル系 table として構造化表示・差分表示の対象にする。
ただし NTF の読み取り仕様では、固定長ファイルはデータ型行の次にフィールド長行を持つ。
ディレクティブ行は固定長ファイル固有の値を持つが、表示・差分仕様は `VARIABLE` と同じ表コンポーネントで扱う。

## 表示原則

ファイル系 table は単なる配列ではなく、ファイル定義とデータを含む表として表示する。

- ディレクティブ行はキーと値を分けて扱い、キー側を構造セルとして灰色系で表示する。
- レコード種別、フィールド定義、型、桁数など、データそのものではなく構造を表すセルは灰色系で表示する。
- 通常のデータセルは標準の表背景で表示する。
- 変更行に含まれる未変更の構造セルは、通常時より少し濃い灰色で表示する。
- 変更された構造セルは、差分色を暗くした色で表示する。
- 変更された通常データセルは、標準の差分色で表示する。

構造セルとは、ユーザーが値データではなくファイル定義として読むべきセルを指す。
固定の語彙だけで判定せず、ブロック内の位置、行の形、周辺行との関係から判断する。

## セクションとレコード種別

セクション名やレコード種別名は任意値として扱う。
セクション判定は、ディレクティブ行の後に現れるファイル定義上のまとまりから行う。

`VARIABLE` / `FIXED` ともに、ディレクティブ行の後に現れるファイル定義上のまとまりをセクションまたはレコード種別として扱う。
その名前はユーザー定義値であり、表示上は構造セルとして扱う。

## 固定長ファイル

`SETUP_FIXED` / `EXPECTED_FIXED` は、`VARIABLE` と同じファイル系 table として扱う。
ただし固定長ファイル固有の意味は失わない。

固定長ファイルで構造として扱うもの:

- ディレクティブ
- レコード種別
- フィールド名称
- データ型
- フィールド長

フィールド名称、データ型、フィールド長の変更は、単なるデータ変更ではなくフィールド定義変更としてレビューできるようにする。

## 列ヘッダと内部キー

ファイル系 table の列ヘッダは空表示にする。
列の追加、削除、並び替えは編集操作 UI で表現する。

内部実装では、名前を持たない配列セルを比較するために位置ベースのキーを使ってよい。
そのキーは比較と更新のための内部識別子として扱い、UI には表示しない。
固定長ファイルも、現行仕様では位置ベースのキーで比較する。

## 差分表示

差分表示は、値の変更だけでなくファイル構造の変更をレビューできることを優先する。

- 追加行、削除行、変更行は行単位で区別する。
- 変更セルはセル単位で強調する。
- 未変更の構造セルは、diff 時も灰色系で表示する。
- 変更された構造セルは、構造セルであることと変更セルであることの両方が分かる暗めの差分色で表示する。
- unified view では、head に対応行がある変更行を before/after 付きの1行として表示する。
- unified view では、行を diff の編集順に従って表示する。
- unified view では、head に対応行がない削除行を、その削除が発生した位置に表示する。
- head に存在する行は head の順序を主表示順にする。
- SCM diff、cell diff、HTML レポートで同じ色の意味を使う。

## 実装ギャップ

現行実装との差分は次の通り。

- `SETUP_FIXED` / `EXPECTED_FIXED` は構造化表示する。
- セクション判定を固定名ではなく構造位置に基づく判定へ移行する。
- unified view のファイル系 table 列ヘッダを空表示にする。
- diff 行でも構造セルの灰色表示を維持する。
- unified の行配置を diff の編集順に従う実装へ移行する。
